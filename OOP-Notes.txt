Polymorphism in OOP (Object-Oriented Programming)

Definition:
Polymorphism means “one name, many forms.” It allows the same method or function to behave differently depending on the object or context.

Key Idea:
A single interface can represent different types of behavior.

Types of Polymorphism

Compile-Time Polymorphism (Static Polymorphism)

Achieved using Method Overloading

Same method name but different parameters

class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

Run-Time Polymorphism (Dynamic Polymorphism)

Achieved using Method Overriding

A child class changes the behavior of a parent class method.

class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }
}

2️⃣ final Method

When a method is declared final,
👉 it cannot be overridden in a child class.

Example:
class Parent {
    final void show() {
        System.out.println("Hello");
    }
}

class Child extends Parent {
    void show() { }  // ❌ Error
}
📌 Why use it?

To prevent changing important logic

To maintain consistent behavior in inheritance

Interview line:

A final method prevents method overriding to protect business logic.

🔹 1. Abstraction in Java
✅ Definition (Write this in exams)

Abstraction is the process of hiding implementation details and showing only essential features of an object.

👉 Focus: What an object does, not how it does it.

🔹 Real-Life Example

When you use an ATM machine, you can:

Withdraw money

Check balance

But you don’t know the internal banking server logic.

That is abstraction.

🔹 How Java Achieves Abstraction?

Java provides abstraction using:

Abstract Class

Interface

🔹 Example using Abstract Class
abstract class Animal {
    abstract void sound();   // abstract method (no body)

    void eat() {             // normal method
        System.out.println("Animal eats food");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }
}
🔹 Key Points

Abstract class → declared using abstract keyword.

Abstract method → no body.

Object of abstract class cannot be created.

Child class must implement abstract methods.

🔹 2. Encapsulation in Java
✅ Definition (Write this in exams)

Encapsulation is the process of wrapping data (variables) and methods into a single unit and restricting direct access to data.

👉 Focus: Data hiding + controlled access.

🔹 How Java Achieves Encapsulation?

Using:

private variables

Public getter and setter methods

🔹 Example
class Student {
    private int marks;   // data hiding

    public void setMarks(int marks) {
        this.marks = marks;
    }

    public int getMarks() {
        return marks;
    }
}
🔹 Why Encapsulation is Important?

✔ Protects data
✔ Improves security
✔ Makes code maintainable
✔ Allows validation before setting values

Example with validation:

public void setMarks(int marks) {
    if(marks >= 0 && marks <= 100) {
        this.marks = marks;
    }
}
🔥 Abstraction vs Encapsulation (Exam Comparison Table)
Feature	Abstraction	Encapsulation
Meaning	Hiding implementation	Hiding data
Focus	What to show	How to protect data
Achieved By	Abstract class / Interface	Private variables + getters/setters
Level	Design level	Implementation level
🎯 Interview-Ready One Line Answer

If interviewer asks:

What is the difference?

You say:

Abstraction hides implementation details and shows only essential features, while encapsulation hides data and restricts direct access using access modifiers.
